How Data Flows Through the Demo

Each audio chunk is processed through a six-stage pipeline that runs automatically as the call progresses.

1

Audio Capture — Triple-Stream Recording

PyAudio captures live microphone input at 44.1kHz mono. Three concurrent streams record simultaneously: a primary stream producing 10-second MP3 segments for real-time processing, a secondary stream offset by 7 seconds to capture words that fall on segment boundaries, and a continuous archive stream for the full uninterrupted recording.

PyAudio pydub MP3 Threading
2

Stage Upload — Snowflake Internal Stage

Each MP3 segment is uploaded to @CALL_CENTER.STG.TRANSCRIPTS using a Snowflake PUT command via the Python connector. Files are named with the pattern {case_id}_{call_id}_{chunk}_{timestamp}_.mp3. The stage uses server-side encryption (SSE), which is required for AI function compatibility.

PUT Command Internal Stage SSE Encryption
3

Transcription — ai_transcribe()

Snowflake's built-in ai_transcribe() function converts each staged MP3 file to text. The result includes the full transcript text and audio duration. Transcriptions are inserted into the CALL_TRANSCRIPTS table with case/call identifiers, chunk numbers, and timestamps parsed from the filename.

ai_transcribe() to_file() INSERT...SELECT
4

AI Extraction — ai_extract()

All transcripts for the current call are aggregated and passed to ai_extract() with a structured response format. The AI identifies nine fields from the conversation: customer name, phone, email, order number, product name, issue description, resolution requested, shipping address, and return reason. New candidate values are de-duplicated and inserted into CALL_CANDIDATE_VALUES.

ai_extract() Hybrid Table 9 Extraction Fields
5

Intelligent Matching — Customer, Product & Case Lookup

Extracted values trigger three parallel lookups: Customer matching searches by name, phone, email, or order number against the CUSTOMERS table. Product matching uses AI_SIMILARITY() to semantically match mentioned products against the customer's actual order history. Similar case detection uses AI_SIMILARITY() to find existing support cases with similar issues — surfacing patterns like recurring product defects across multiple customers.

AI_SIMILARITY() Hybrid Tables CASES Table Semantic Search
6

Real-Time UI Update — WebSocket Push

All results are pushed to the React frontend via WebSocket in a single message containing extracted candidates, matched customers, similar cases, and identified products. The UI updates instantly — the agent sees customer details, order history, product matches from their orders, and a panel of similar cases with resolutions, all without lifting a finger.

Snowflake AI Functions in Action

Three Cortex AI functions power the entire intelligence pipeline — no external APIs, no data movement, no infrastructure to manage.

🎙️

ai_transcribe()

Converts staged MP3 audio files to text directly inside Snowflake. Returns transcript text and audio duration metadata. Runs as a SQL function — no external speech-to-text service needed.

SELECT ai_transcribe(to_file(@stage, 'audio.mp3'))
🧠

ai_extract()

Extracts structured fields from unstructured transcript text using a JSON response format. Pulls customer names, phone numbers, order numbers, product mentions, issue descriptions, and resolution requests from natural conversation.

SELECT ai_extract(text=>transcript, responseformat=>...)
🔍

AI_SIMILARITY()

Performs semantic similarity matching between text strings. Used to match mentioned product names against the catalog, and to find existing support cases with similar issue descriptions — enabling pattern detection across customers.

WHERE AI_SIMILARITY(issue_text, case_desc) > 0.4

Real-Time Agent Assistance

As the conversation unfolds, the application automatically surfaces relevant information to help the agent resolve issues faster.

👤

Auto-Customer Identification

When the caller mentions their name, phone number, or email, the system automatically searches the customer database and displays their full profile including loyalty tier, address, and complete order history with line-item details.

📦

Product Match from Order History

When a product is mentioned in conversation, the system uses semantic matching to identify the exact product from the customer's order history — showing the specific order it was on, order status, date, and tracking number.

⚠️

Similar Case Detection

As the issue description emerges from conversation, the system searches all historical cases for similar problems. Surfaces a ranked list with similarity scores, showing other affected customers, resolutions applied, and whether a pattern (like a product defect) exists.

Demo Scenario: The Headphone Defect Pattern

The sample data includes a compelling scenario that demonstrates the full power of the system.

📞

A customer calls about their headphones

The caller identifies themselves and describes an issue: "The left ear on my noise-canceling headphones keeps cutting out." The system transcribes this in real-time and extracts the customer name, product mention, and issue description.

👤

Customer identified, order history loaded

The system matches the caller to their customer record and loads their order history. The "Wireless Noise-Canceling Headphones" are semantically matched against their actual purchase, showing the specific order, date, and delivery status.

🚨

Similar cases alert fires — a pattern emerges

The Similar Cases panel lights up: 5 other customers have reported the same left-ear audio dropout on the same product. Two cases are already closed with resolutions ("Replacement unit shipped. Known firmware issue."), and three remain open with escalating priority from HIGH to CRITICAL. The agent immediately understands this is a known defect — not a one-off issue.

✅

Agent resolves with confidence

Armed with the pattern data and previous resolutions, the agent can proactively offer a replacement, reference the known firmware issue, and resolve the call faster — turning a frustrating experience into a demonstration of excellent support.
